# Canonical Expedition Example

This fictional context demonstrates how a project can preserve shared domain language for developers and agents. Replace these terms with the application's real vocabulary when removing the canonical example.

## Language

**Mission Brief**:
The validated description of a proposed expedition, including its call sign, destination, crew, duration, and purpose.
_Avoid_: Form data, request payload

**Expedition Plan**:
An immutable, transient planning result generated from a Mission Brief.
_Avoid_: Result, response

**Supply Recommendation**:
Equipment guidance for one of the navigation, survival, or mission-specialist concerns of an Expedition Plan.
_Avoid_: Item, suggestion

**Risk Classification**:
The Routine, Elevated, or Extreme risk level calculated from a Mission Brief.
_Avoid_: Readiness, status

**Destination**:
The Ember Moon, Glass Nebula, or Tidal Archive location selected for an expedition.
_Avoid_: Location option

**Mission Purpose**:
The Survey, Research, or Rescue objective selected for an expedition.
_Avoid_: Type, category
